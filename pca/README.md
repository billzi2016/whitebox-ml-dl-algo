# PCA 逐步求主成分动画

这个目录演示 PCA 的算法过程，不调用 `sklearn.decomposition.PCA`。

![PCA 主成分迭代过程演示](pca-demo.png)

当前实现手写：

- 数据中心化。
- 协方差矩阵。
- 幂迭代求第一主成分。
- 正交化后求第二主成分。
- 三维点云到二维主成分坐标的投影。

## 1. 运行方式

```bash
python3 pca/main.py
```

可调参数：

```bash
python3 pca/main.py --init-seed 9
python3 pca/main.py --steps-per-frame 2
```

## 2. 数据是什么

数据是一个三维相关点云。它先从潜在二维结构生成，再旋转到三维空间，并加入少量离群点。

这样做的目的：

- 原始坐标轴不等于主成分方向。
- 数据有明显最大方差方向。
- 二维投影能展示 PCA 的降维效果。

## 3. 中心化

PCA 先减掉均值：

\[
\tilde{x}_i = x_i - \bar{x}
\]

如果不中心化，第一主成分会受到数据整体位置影响，而不是只反映方差方向。

## 4. 协方差矩阵

协方差矩阵是：

\[
\Sigma = \frac{1}{n-1}X^\top X
\]

其中 \(X\) 是中心化后的数据矩阵。

PCA 要找的第一主成分是让投影方差最大的方向：

\[
v_1 = \arg\max_{\lVert v\rVert=1} v^\top \Sigma v
\]

## 5. 幂迭代

幂迭代用来逐步逼近最大特征值对应的特征向量：

\[
v \leftarrow \Sigma v
\]

\[
v \leftarrow \frac{v}{\lVert v\rVert}
\]

动画里黑色方向线就是当前第一主成分。它会从随机方向逐步旋转到最大方差方向。

## 6. 第二主成分

第二主成分要和第一主成分正交。代码先做：

\[
u = \Sigma v_2
\]

再减掉第一主成分方向：

\[
u \leftarrow u - (u^\top v_1)v_1
\]

最后单位化：

\[
v_2 = \frac{u}{\lVert u\rVert}
\]

## 7. 动画每帧表示什么

每一帧执行若干次幂迭代：

1. 更新第一主成分方向。
2. 更新第二主成分方向并正交化。
3. 把三维点投影到当前 PC1/PC2。
4. 显示三维中的方向线和右侧二维投影。

标题里的 `var1` 和 `var2` 是当前两个方向上的解释方差：

\[
v^\top \Sigma v
\]
